# Projet de Problem Solving 

Implémenter obligatoirement  en C un simulateur permettant de comparer les algorithmes de Tri suivants (on considère le tri dans l'ordre croissant) :  tri sélection, tri à bulles, tri par insertion, tri Shell et tri rapide.
La comparaison se fait à travers la comparaison du temps d'exécution lorsque on varie le nombre d'entiers N dans le tableau. Le résultat est une courbe montrant l'évolution du temps d'exécution en fonction de N ( peut atteindre 10 Millions d'éléments et plus si possible).

    un tableau non trié (remplissage aléatoire).
    un tableau déjà trié,
    un tableau trié en ordre inverse,

Un rapport descriptif ne dépassant pas 10 pages devra être rédigé (dans rapport.md (à transformer plus tard en pdf)). Le rapport doit contenir une explication du comportement des algorithmes de Tri étudiés, le code permettant de faire l'étude, la configuration de la machine utilisée et les courbes générées.
